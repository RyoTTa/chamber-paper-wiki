# APT: Securing Against DRAM Read Disturbance via Adaptive Probabilistic In-DRAM Trackers

**Venue:** ASPLOS 2026
**저자:** Runjin Wu, Meng Zhang*, You Zhou, Changsheng Xie, Fei Wu (Huazhong University of Science and Technology)

## 개요

DRAM scaling은 high-capacity를 가능하게 했으나 cell size와 inter-cell spacing을 감소시켜 **DRAM read disturbance**를 심화시킴: aggressor row에 대한 activation이 인접한 victim row의 data integrity를 훼손.

두 가지 주요 disturbance 현상:
- **RowHammer [25]**: Aggressor row의 반복적인 activation/precharge. TRH (RowHammer Threshold)가 DDR3의 139K에서 LPDDR4의 4.8K까지 급감, 미래에는 sub-1K로 예상.
- **RowPress [29]**: Aggressor row를 extended duration open 상태로 유지. RowHammer보다 activation당 disturbance가 최대 2 order of magnitude 더 강함. tON > tRAS에서 row open time을 길게 유지.

**In-DRAM mitigation의 장점**: DRAM chip 내부에서 transparent하게 해결, vendor-specific TRH tuning 가능. 그러나 두 가지 핵심 과제:

1. **Row-Sampling Miss (RSM)**: DDR5 도입된 Refresh Management (RFM) command에 의존하는 probabilistic tracker들은 fixed-probability sampling을 사용. RowPress가 window 내 적은 activation을 유발하면 sampling probability 부족으로 attack row가 mitigation을 회피 (Figure 1c). 예: MINT는 window size W_A=75에서 TRH=1419이나, activation이 4개만 있는 RowPress attack 시에도 TRH=1262 → 실질적 보호 실패.

2. **RFM의 보안 취약성**: RFM-induced latency spike (tRFMsb=190ns)가 timing side/covert channel로 악용 가능 [5, 34, 50, 56]. TB-RFM [56]은 fixed interval로 RFM을 발행하여 timing leakage를 제거하나, TRH≤512에서 25% slowdown (Figure 4).

**Goal**: Sub-1K TRH를 견디는 low-cost, secure probabilistic in-DRAM tracker 개발, RFM에 의존하지 않음.

## 방법론

### 1. APT: Reservoir Sampling 기반 적응형 tracker

핵심 아이디어: **Reservoir Sampling algorithm [53]** — item stream에서 K개 item을 uniform probability로 sampling. APT는 tREFI 내 activation stream에 적용.

**Operation**:
1. **Initialization**: 첫 K개 activation은 직접 reservoir에 삽입.
2. **Online Update**: i번째 activation (K+1 ≤ i ≤ N)에 대해 probability K/i로 선택, 선택 시 reservoir 내 random item 교체.

K=1인 single-entry APT 기준:
- Sampling probability per activation: P_sample = 1/i
- Retention probability (tREFI 종료까지 유지): P_retention = ∏_{j=i+1}^{N} (1 - 1/j)
- **Eventual equal mitigation probability**: P_mitigation = P_sample · P_retention = 1/N

**핵심 성질**: ∑_{i=1}^{N} P_mitigation = 1 — 항상 최소 1개의 activation이 sampling됨 → **RSM 완전 제거**.

### 2. Hardware implementation (Figure 6)

7개 component로 구성 (per bank):

| Component | 역할 | Size |
|-----------|------|------|
| AC (Activation Counter) | Activation에 sequence number 부여 (1 ≤ i ≤ W_A) | 7-bit |
| SMPC-1 | First activation 감지 (AC==1?) | 1 comparator |
| RNR (Random Number Register) | tREFI window용 random number 저장 | 14-bit |
| MSS (Multi-Stage Subtractor) | Staged subtraction, operand width W 증가 | 7→14 stages |
| SUBC (Subtraction Comparator) | MSS output < i 인지 확인 | 1 comparator |
| SMPC-0 | Modulo result == 0 감지 | 1 comparator |
| MAR (Mitigation Address Register) | Sampled row address 저장, valid bit 포함 | 18-bit |

**Multi-Stage Subtractor timing**: 식 (5)에 의해 operand width 증가 시점 결정:
- C > (2^W - 1)·tCK / T_s = (2^W - 1)·0.625ns / 40ns
- 도출된 thresholds: C=4, 8, 16, 32, 64, 128, 256 for W=8, 9, 10, 11, 12, 13, 14

Subtraction latency는 tRC(46ns)에 완전히 overlapped.

**Flow per activation** (tREFI window):
1. AC→i 할당
2. i=1이면 address를 MAR에 저장 후 종료; 아니면 MSS+SUBC로
3. RNR forwarded to SUBC
4. MSS가 staged subtraction 수행
5. SUBC: output < i 이면 valid modulo-i result. result == 0 이면 activation sampled → MAR에 저장

**REF 시**: Valid MAR 있으면 Step Mitigation으로 victim rows refresh. AC reset, MAR invalidate, RNR 재생성.

### 3. Step Mitigation (SM): Transitive attack 방어

6 tRC intervals available at REF (tRFC=410ns, tRC=46ns). 기존 BR=2 transitive mitigation은 2 victim refresh 필요 → REF당 1 mitigation만 가능.

**Step Mitigation의 원리**:
- Aggressor-victim distance별로 group화, step-wise decreasing refresh probabilities 할당 (Figure 10).
- Reservoir Sampling의 변형: distance sampling model.
- Step i: width w_i (distance 그룹 크기), height h_i (이전 step 대비 probability drop factor).
- Sampling probability: P_smp_il = (h_i/P_{i-1} + l)^{-1}
- Refresh probability per distance at step i: Pref_i = 1/(h_{i-1}·P_{i-1} + w_i) · ∏_{j=i+1}^{n} (h_{j-1}·P_{j-1})/(h_{j-1}·P_{j-1} + w_j)

**Conservative configuration** (characterization study [27] 기반):
- n=3 steps, w_1=1, h_1=62, w_2=1, h_2=2, w_3=2 (BR=4)
- Pref_1 = 31/32 (distance 1), Pref_2 = 1/64 (distance 2), Pref_3 = 1/128 (distance 3), Pref_3 = 1/128 (distance 4)

**Sum guarantee**: Σ w_i · Pref_i = 1 — 항상 하나의 distance가 victim refresh로 선택됨.

**SM security analysis** (Figure 18, 4 attack patterns):
- Pattern-1 (proximate to aggressor): NMR ≈ 41.45·P/[-h_1·ln(1-P)] ≤ 20 (negligible for h_1 ≥ 2)
- Pattern-2 (among distances at same step): NMR = 2·P·M → TRH ≤ 41
- Pattern-3 (before step drop): TRH = 20.73·(1+1/h_i) ≤ 31
- Pattern-4 (after step drop): TRH = 20.73·(1+h_i) ≥ 62

SM은 APT designs with TRH ≥ 67에 대해 secure (imprecision 포함 시 66→67).

## 핵심 기여

- **APT**: Reservoir Sampling을 DRAM in-chip tracker에 최초 적용, real-time activation count 기반 adaptive uniform mitigation probability → RSM 완전 제거.
- **Step Mitigation**: 2개의 victim refresh만으로 transitive attack을 방어하는 최초의 victim-refresh strategy. Device-specific charge-loss profile에 정확히 matching. APT+SM = worst-case TRH 694 with 0% slowdown.
- **APT+TB-RFM**: TRH 228까지 확장, 1.6% slowdown + 13.5 bytes/bank storage. TPRAC 대비 0.15× slowdown at TRH=256.
- **Broader significance**: 미래 sub-1K TRH DRAM 세대를 위한 실용적이고 scalable한 in-DRAM defense. RFM 보안 취약성을 우회하며 극도로 낮은 hardware cost.

## 주요 결과

3개의 independent single-entry reservoir sampler를 round-robin으로 운영 (Figure 12).
- Modulo-3 counter: 각 sampler가 W_A/3 = 25 activation capture.
- P_mitigation ≥ 1/25 → worst-case TRH = 490 (imprecision 2.2%).
- 3-entry buffer per bank, 각 REF에서 3개 mitigation 수행 (=6 victim refreshes).

**APT-P**: 최대 15 entries per bank, refresh postponement 대응 (Appendix C).
- 최대 postponement interval: 5·tREFI (415 activations). T_ACT = 84.
- Worst-case TRH = 694 (imprecision 포함).
