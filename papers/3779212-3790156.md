# Efficient Remote Memory Ordering for Non-Coherent Interconnects

**Venue:** 
**저자:** Wei Siew Liew, Md Ashfaqur Rahaman, Adarsh Patil, Ryan Stutsman, Vijay Nagarajan (University of Utah, Arm)

## 개요

PCIe와 같은 non-coherent interconnect는 오늘날 서버의 CPU-NIC 통신을 지배한다. 그러나 PCIe specification은 fine-grained memory ordering을 end-to-end로 표현·강제할 수 있는 semantics가 부족하여, 소프트웨어는 source-side serialization이라는 값비싼 해결책에 의존해야 한다.

구체적으로 두 가지 치명적 bottleneck이 존재한다:

1. **CPU→NIC 패킷 전송 (W→W MMIO ordering):** CPU가 NIC 메모리에 MMIO write로 패킷을 전송할 때, 패킷 순서를 보장하려면 매 패킷마다 `sfence`를 삽입해야 한다. 실측 결과, 64B 패킷에서 sfence 없이는 122 Gb/s throughput이 나오지만, sfence를 넣으면 89.5% 감소하여 100 Gb/s link를 전혀 활용하지 못한다 (Figure 4). 이 때문에 현대 시스템은 MMIO doorbell + DMA read라는 복잡한 우회 경로를 사용한다.

2. **RDMA KVS lookup (R→R DMA ordering):** NIC가 RDMA READ로 lock flag → data 순서로 읽어야 하는 "check-before-read" 패턴에서, PCIe는 read 간 ordering을 보장하지 않는다. NIC는 첫 번째 read의 full round-trip 완료(≈500 ns)를 기다린 후 두 번째 read를 발행해야 하며, 이는 throughput을 ~2 Mops/s로 제한한다. ConnectX-6 Dx 실측에서 ordered DMA read는 unordered 대비 RTT당 약 300 ns 추가 지연을 발생시킨다 (Figure 2). RDMA READ pipelining 시 ordered read는 5.0 Mop/s (2.37 Gb/s), unordered write는 3배 높은 throughput을 보인다 (Figure 3).

핵심 통찰: **문제는 source(CPU/NIC)에서 ordering을 강제하기 때문**이다. destination(Root Complex)에서 ordering을 책임지면 source-side stall을 제거할 수 있다.

## 방법론

### 1. PCIe Extensions: Acquire/Release Semantics 도입

ARM/RISC-V ISA의 memory consistency model에서 영감을 받아, PCIe TLP header에 acquire/release bit를 도입한다:

- **PCIe Acquire Read:** 기존 Relaxed Ordering bit를 재활용하거나 새 bit 추가. 이 bit가 set된 read는 "이후 모든 memory access가 이 read의 결과를 관찰해야 함"을 의미.
- **PCIe Release Write:** Relaxed Ordering bit를 release로 reinterpret. "이전 모든 action이 다른 agent에게 visible해진 후 이 write 수행."

예시: NIC가 flag(DMA read, acquire) → data(DMA read, relaxed) 순서로 발행. Acquire가 완료되기 전까지 data read의 결과가 NIC에 return되지 않도록 RC에서 보장.

### 2. Host ISA Extensions: First-Class MMIO Instructions

기존 x86/RISC-V ISA는 MMIO와 local memory access를 구분하지 못한다. 이를 해결하기 위해 4가지 새로운 instruction을 제안:

- **MMIO-Store / MMIO-Load:** 기본 MMIO 접근.
- **MMIO-Release:** "이전 모든 host memory operation이 visible해진 후 MMIO write 수행."
- **MMIO-Acquire:** "MMIO read 완료 후에만 이후 host memory operation 수행."

이 instruction들은 memory ordering을 CPU→interconnect 경계에서 명시적으로 표현하며, fence를 "stall"이 아닌 "ordering metadata 주입 지시자"로 재해석한다.

### 3. Remote Load-Store Queue (RLSQ) — Root Complex Microarchitecture

#### 3.1 Baseline RLSQ

- DMA write: PCIe write는 strongly-ordered이므로 FIFO 순서로 serialize. 단, coherence request(invalidation 등)는 병렬 발행 가능.
- DMA read: PCIe read는 weakly-ordered이므로 병렬 dispatch.

#### 3.2 Release-Acquire RLSQ

새로운 semantics를 충실히 구현:

- **PCIe Acquire 도착:** 이후 모든 request의 issue를 block, 자신의 coherent request가 완료될 때까지 대기.
- **PCIe Release 도착:** 이전 모든 request 완료까지 대기 후 자신의 request issue.

이 semantics는 NIC가 lock acquire read + speculative data read를 pipelining하는 aggressive한 디자인을 안전하게 만든다. RC에서 acquire 완료 전까지 speculative data read를 stall시켜 memory consistency를 보장.

#### 3.3 Optimization 1: Thread-specific Ordering (Per-Thread RLSQ)

Global하게 ordering을 강제하면 서로 다른 thread(context)의 독립적인 request가 false dependency에 걸린다. 해결: PCIe TLP에 thread ID 필드를 추가하고, RLSQ가 thread ID별로 acquire/release를 추적. Microarchitecture적으로는 단일 RLSQ를 logically partition하여 "awaiting acquire" flag만 per-thread로 관리.

#### 3.4 Optimization 2: Speculative DMA Ordering (RC-opt)

RLSQ가 out-of-order processor의 speculative load처럼 동작:

- Acquire→Read 시퀀스에서 두 request를 **병렬**로 host memory에 발행.
- Speculative read의 결과를 buffer에 저장하고, acquire 완료 후 NIC에 순서대로 전달.
- Host write에 의한 coherence invalidation을 snoop: speculative read 주소에 write가 발생하면 해당 read만 squash 후 retry. LSQ와 달리 **충돌난 read만** squash (후속 operation은 유지).

동일 원리가 Write→Release에도 적용: data write의 coherence 작업과 release를 병렬 수행.

## 핵심 기여

**핵심 contribution:** PCIe의 remote memory ordering을 "fabric의 implicit property"에서 "ISA와 co-design된 first-class semantic"으로 승격. Destination-based ordering model을 통해 source-side serialization을 제거.

**주요 결과:**
- MMIO W→W: fence-free direct transmit path로 100 Gb/s line rate 달성.
- DMA R→R: speculative ordering으로 unordered와 동일한 throughput 제공 (최대 50.9× 향상).
- 새로운 Single Read KVS protocol이 이전에는 불가능했으나, 본 ordering 지원으로 FaRM 대비 1.6× throughput + 단순성 확보.
- HW overhead: 0.85% area, 0.54% static power 증가로 실용적.

**Broader significance:** CXL과 같은 coherent interconnect가 과연 필요한지에 대한 근본적 질문을 제기. Non-coherent interconnect도 올바른 ordering abstraction만 있으면 고성능 host-device communication에 충분할 수 있다.

## 주요 결과

MMIO instruction에 sequence number를 부여하고, RC(또는 device endpoint)의 ROB가 순서를 재구성:

- CPU가 MMIO-Store(#N) → MMIO-Release(#N+1) 발행.
- RC에 #N+1이 #N보다 먼저 도착하면, #N 도착까지 #N+1의 PCIe write를 지연.
- Thread ID를 sequence number에 포함시켜 per-thread 독립적 관리.

ROB를 endpoint에 배치하면 interconnect 전체 구간에서 unordered PCIe read/write를 사용할 수 있어 utilization이 극대화.

## 평가

### Methodology

| 항목 | 내용 |
|------|------|
| Simulator | gem5 (classic cache model) |
| DMA 실험 CPU | SimpleTimingCPU, 3 GHz |
| MMIO 실험 CPU | O3CPU, 3 GHz |
| I/O bus latency | 200 ns (one-way) |
| RC latency | 17 ns (DMA), 60 ns (MMIO) |
| RLSQ entries | 256 |
| HW validation | NVIDIA ConnectX-6 Dx 100 Gb/s, CloudLab sm110p (Xeon Silver 4314) |
| Benchmarks | Ordered DMA Reads, KVS Gets (Validation/Single Read/FaRM), NIC Packet Transmission |

### DMA Read Throughput (Figure 5)

- **NIC ordering:** 각 cache line마다 synchronous stall → throughput collapse (작은 size에서 수 GB/s).
- **RC ordering:** stall을 RC로 이동 → ~5× 향상. 하지만 여전히 sequential access bottleneck (≈100 ns/read).
- **RC-opt (speculative):** unordered와 동일한 throughput 달성. 모든 크기에서 100 Gb/s NIC limit 포화.

### KVS Get Throughput (Figure 6)

Validation protocol (2 RDMA READs per get) 기준:

- **Single QP, batch=100:** NIC=baseline, RC=29.1×, RC-opt=**50.9×** 향상 (64B objects).
- **Multi-QP scaling (1-16 QPs):** NIC는 16 QPs까지 확장해도 RC-opt에 수렴하지 못함. ConnectX-6 Dx는 16 QP 이상에서 RDMA READ pipelining이 크게 확장되지 않기 때문.
- **16 QP, batch=500:** RC-opt만이 100 Gb/s link에 근접. 작은 object size에서도 correctness와 throughput을 모두 유지.

### Real NIC Emulation (Figure 7)

ConnectX-6 Dx에서 read-only workload 기준으로 unordered = speculative ordered의 throughput이 동일함을 검증 후, 4가지 algorithm 비교 (16 threads, batch=32):

| Algorithm | 특징 | 64B throughput |
|-----------|------|----------------|
| Pessimistic | RDMA fetch-and-add로 lock | 매우 낮음 (작은 size에서 overhead 큼) |
| FaRM [NSDI '14] | Cache line마다 version number embedding | Version metadata stripping overhead로 제한됨 |
| Validation | 2 RDMA READs + version check | 60+ Gb/s @512B. 단, R→R ordering 필요 |
| **Single Read** | Header/footer version만 확인, 1 RDMA READ | **Validation 대비 2×, FaRM 대비 1.6×** 향상 (64B) |

Single Read protocol은 PCIe read reordering 때문에 이전에는 unsafe했으나, 본 논문의 ordering 지원으로 가능해졌다. FaRM보다 단순하고 metadata copy overhead가 없다.

### MMIO Write (Figure 10)

Simulation에서도 HW 결과와 동일: fence 사용 시 throughput 급감. 본 논문의 sequence number + ROB 접근은 fence 없이도 ordering 보장.

### HW Overhead (Table 5, 6)

CACTI 65nm 모델:

| Component | Area (mm²) | I/O Hub 대비 | Static Power (mW) | I/O Hub 대비 |
|-----------|-----------|-------------|-------------------|-------------|
| RLSQ (256-entry, fully-assoc) | 0.9693 | 0.685% | 49.20 | 0.492% |

### Peer-to-Peer Experiments (Figure 9)

P2P transfer 시 HOL blocking 문제 분석:

- **P2P-noVOQ:** congested P2P device로 인해 CPU flow까지 167× 성능 저하 (8192B).
- **P2P-VOQ:** Virtual Output Queue로 flow isolation → baseline 수준 throughput 회복.
