# PIPM: Partial and Incremental Page Migration for Multi-host CXL Disaggregated Shared Memory

**Venue:** 
**저자:** 

## 개요

CXL 3.x 표준은 multi-host cache-coherent disaggregated shared memory(CXL-DSM)을 지원하여 여러 host가 CXL memory pool을 coherent하게 공유 가능. 하지만 remote CXL memory latency는 local DRAM 대비 2~3배 느림 [48, 55, 94].

기존 page migration 기법(Nomad [90], Memtis [45], HeMem [68])은 single-host CXL disaggregated memory용으로 설계되어 multi-host CXL-DSM에서 두 가지 한계 노출:

**(1) Local gain, global pain.** Single-host에서 hot page를 local DRAM으로 옮기면 항상 이득이지만, multi-host CXL-DSM에서는 한 host로 page를 migration하면 다른 host들은 해당 page를 non-cacheable로 접근해야 함 (Figure 3). Inter-host 접근은 4-hop traversal이 필요하고, CXL memory 접근은 2-hop으로 충분. 정량 평가 결과 Nomad의 34%, Memtis의 29%가 harmful migration으로 전체 실행시간을 증가시킴 (Figure 5).

**(2) Poor migration scalability.** Multi-host CXL-DSM은 더 짧은 migration interval을 요구하나 (여러 host의 access pattern을 빠르게 반영해야 함), 1ms interval에서 Nomad는 26.1%, Memtis는 15.4% 실행시간 증가 (Figure 4). Migration overhead가 page transfer + page-table update + TLB invalidation에서 발생.

기존 기법들이 실패하는 근본 원인은 **single-destination, rigid per-page migration**. 전체 page를 하나의 host로 완전히 옮기거나 CXL memory에 그대로 두는 이분법적 결정만 가능. 동일 page 내에서도 cache block별로 access pattern이 다른 점을 활용하지 못함.

## 방법론

PIPM은 hardware-based solution으로, OS 수정 없이 투명하게 동작. 두 가지 핵심 아이디어:

- **Partial Migration:** page 전체가 아닌, host가 자주 접근하는 cache block만 선택적으로 local DRAM으로 이동. 자주 접근되지 않거나 다른 host가 접근하는 block은 CXL memory에 유지. 이로써 local/inter-host access pattern을 fine granularity로 구분.
- **Incremental Migration:** 명시적 page transfer 대신, 프로그램의 intrinsic memory access(cache eviction/writeback)를 활용하여 cache block을 점진적으로 이동. 추가 data transfer overhead 없음.

### 1. PIPM Majority-vote Migration Policy

Boyer-Moore majority vote algorithm [11]에서 영감을 받은 hardware-based migration policy.

**데이터 구조:**

| Component | Location | Entry size | Fields |
|---|---|---|---|
| Global Remapping Table | CXL memory node | 2 Bytes | 5-bit current host ID, 5-bit candidate host ID, 6-bit global counter |
| Global Remapping Cache | CXL device | 16KB, 8-way | Cache for global remapping entries |
| Local Remapping Table | 각 host DRAM | 4 Bytes | 28-bit local PFN (1TB indexing), 4-bit local counter |
| Local Remapping Cache | 각 host RC | 1MB, 8-way | Cache for local remapping entries |

**동작 알고리즘:**

1. Global counter가 현재 candidate host로부터의 접근 시 +1, 다른 host 접근 시 -1.
2. Global counter가 0에 도달하면 다음 접근 host가 새 candidate host로 설정됨 ①.
3. Global counter가 predefined migration threshold (기본값=8)에 도달하면 partial migration 개시 ②: candidate host의 local remapping table에 entry 생성, local PFN 할당 (OS/hypervisor), local counter를 threshold로 초기화 ③.
4. Partial migration 완료 후 current host ID가 해당 host로 설정됨. 이후 local 접근은 local counter만 감소 (global counter 우회) ④.
5. Inter-host 접근은 local counter를 decrement ⑤. Local counter가 0이 되면 partial migration revoke: 모든 cache block을 CXL memory로 이동, local remapping entry 제거, global remapping entry reset ⑥.

**핵심:** Partial migration 개시는 remapping table update만 수행 (page-table update, TLB invalidation 불필요). 실제 data 이동은 incremental migration에 위임.

### 2. PIPM Coherence Protocol

기존 multi-host CXL-DSM에서 host local memory는 coherence domain 밖에 있어 partially migrated cache block에 대한 coherent/cacheable 접근이 불가능. PIPM coherence는 이를 해결.

**추가된 상태:**

- **ME (Migrated-Modified/Exclusive):** cache block이 local memory로 migration되었고 해당 host cache에 exclusively 존재. Local coherence directory에 ME state + in-memory bit=1.
- **I' (Migrated-Invalid):** cache block이 local memory로 migration되었으나 cache에는 없음. Local/device directory에 I state + in-memory bit=1.

**Coherence state encoding:**

| | Host Side | | Device Side | |
|---|---|---|---|---|
| Dir. State | Mem. State | PIPM State | Dir. State | Mem. State | PIPM State |
| M/S/I | 0 | M/S/I | M/S/I | 0 | M/S/I |
| ME/I | 1 | ME/I' | I | 1 | I' |

**6가지 핵심 transition (Figure 9):**

- **Case ① Loc-WB → Incremental Migration:** Local directory state=M이고 아직 migration 안 된 block에 writeback 발생 시. CXL→local로 incremental migration: 양쪽 in-memory bit flip (0→1), directory state M→I'로 전환.
- **Case ② Inter-Rd/Wr in I':** 다른 host가 migrated block(I' state)에 접근 시. CXL directory가 owning host의 local directory에 요청 → data + in-memory bit 회수 → asynchronous memory writeback으로 CXL memory 복원, state I'→I.
- **Case ③④ Local Accesses:** 이미 migration된 block에 대한 local read/write. Local memory에서 직접 서비스, host directory I'→ME. CXL directory entry 불필요 (traffic 감소).
- **Case ⑤⑥ Inter-Rd/Wr in ME:** ME state block에 대한 inter-host 접근. Owning host directory: Inter-Wr → ME→I (⑤), Inter-Rd → ME→S (⑥). Asynchronous memory writeback으로 CXL memory 복원.

**Strawman (Naive Coherence)와 비교:** 1-bit in-memory state만 추가하는 단순 접근은 모든 요청이 CXL device directory를 거쳐야 하며 불필요한 CXL link round-trip 발생. PIPM coherence는 local access를 CXL directory 우회하여 처리.

### 3. Two-Level Remapping Table

Local remapping table은 two-level radix page table로 구성 (32MB fixed root node, 4B/4KB × RSS). 전체 RSS의 약 0.1% 공간 사용. Global remapping table은 CXL-DSM의 0.05%만 사용.

Remapping table lookup은 shared data access 시에만 필요. Local private data는 기존 physical address range check로 구분되어 PIPM overhead 없음.

## 핵심 기여

**Core contributions:**

1. Multi-host CXL-DSM 환경에서 page migration의 두 가지 근본적 한계(local gain/global pain, poor scalability)를 정성적·정량적으로 규명.
2. **Partial + Incremental Page Migration**이라는 새로운 패러다임 제시: cache block granularity 선택적 migration + intrinsic memory access 활용 data movement.
3. Hardware-based majority-vote migration policy로 multi-host 환경에서 globally optimal한 migration 결정.
4. PIPM coherence protocol로 local memory를 coherence domain에 통합, incremental migration과 cacheable inter-host access 동시 지원.
5. 평균 1.86× speedup, 최대 2.54×, 56.1% local hit rate, 1.5% inter-host stall overhead.

**Broader significance:** CXL 3.x 기반 rack-scale disaggregated memory system에서 page migration의 실용성을 최초로 입증. Application-level optimization(SW prefetching, lock-free data access)과 orthogonal하게 통합 가능. 향후 CXL-DSM coherence directory의 on-die area overhead 절감에도 기여.

## 주요 결과

4-host CXL-DSM, migration threshold=8, graph analytics workload (SSSP):

1. Host-0이 page P를 8회 연속 접근 → global counter=8 → threshold 도달.
2. Partial migration 개시: Host-0의 local remapping table에 entry 생성 (CXL PFN `0x5f7...` → local PFN `0x237...`), local counter=8로 초기화.
3. Host-0의 후속 writeback이 발생할 때마다 해당 cache block이 incremental하게 local DRAM으로 이동 (Case ①).
4. Host-1이 page P 접근 → inter-host access → local counter 8→7. CXL directory를 통해 Host-0의 local directory에서 data fetch (Case ②).
5. Host-0이 계속 접근: local counter 유지 (④). Host-1 접근이 누적되어 local counter=0 → partial migration revoke, 모든 migrated block CXL memory로 복원.
