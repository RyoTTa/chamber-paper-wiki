# A Programming Model for Disaggregated Memory over CXL

**Venue:** ASPLOS 2026
**저자:** Gal Assa (Technion), Moritz Lumme (ETH Zürich), Lucas Bürgi (ETH Zürich), Michal Friedman (ETH Zürich), Ori Lahav (Tel Aviv University)

## 개요

CXL (Compute Express Link)은 여러 cloud hyperscaler가 개발에 참여하는 차세대 데이터센터 interconnect 표준으로, cache-coherent shared memory pool을 disaggregated fashion으로 가능하게 함. CXL은 load/store instruction만으로 모든 유형의 memory (DRAM, SSD 등)를 unified shared main memory처럼 접근할 수 있게 함.

그러나 CXL의 등장과 함께 두 가지 가정이 깨짐:

1. **Heterogeneous compute units** (CPU, GPU, FPGA, accelerator)가 같은 memory를 공유하므로 single memory model로 correctness를 논할 수 없음.
2. **Single failure domain이 아님** — node가 독립적으로 crash 가능. Compute node crash 시 memory server의 data는 살아있을 수 있고, memory server crash 시 compute node의 cached data가 유실될 수 있음.

기존 persistent memory transaction 연구는 full-system crash model을 가정하며, CXL의 partial crash model에 그대로 적용 불가능. **CXL은 아직 general programming model이 없어**, consistency guarantee가 무엇인지, local memory model이 CXL fabric 위에서 보존되는지 불분명함.

## 방법론

### 1. CXL0 primitive abstraction

CXL specification (CXL 4.0)의 low-level transaction들을 single abstract read와 3종의 store, 2종의 flush로 추상화:

**Load**: Cache coherence 가정 하에 단일 primitive. Cache에 valid value가 있으면 거기서 읽고, 없으면 memory owner의 physical memory에서 읽음.

**3종의 Store**:
- **LStore (Local Store)**: Local cache에만 write. CXL protocol은 언제 physical memory로 propagate될지 보장하지 않음. 가장 빠름.
- **RStore (Remote Store)**: Address owner의 cache 또는 physical memory에 도달하면 완료로 간주. Non-owner가 remote address에 write할 때 owner의 cache에 기록됨.
- **MStore (Memory Store)**: Physical memory에 도달해야만 완료. 가장 강한 durability 보장, 가장 느림.

**2종의 Flush**:
- **LFlush (Local Flush)**: Local cache line을 다음 hierarchy (remote cache 또는 local memory)로 write-back. Blocking semantics — C_i(x)=⊥ precondition으로 구현.
- **RFlush (Remote Flush)**: 모든 cache에서 해당 cache line을 제거하고 owner memory로 write-back. ∀j. C_j(x)=⊥ precondition.

**GPF (Global Persistent Flush)**: 전체 address space에 대한 RFlush와 동등. Optional이며 planned shutdown/reboot 용도.

**Non-deterministic propagation**: Cache replacement policy에 의한 cache-to-cache (horizontal), cache-to-memory (vertical) propagation.

### 2. Formal operational semantics (CXL0)

System state: γ = (C, M), where:
- C_i: Loc → Val ⊎ {⊥} (machine i의 cache)
- M_i: Loc_i → Val (machine i의 physical memory)

**Global invariant**: ∀i,j. ∀x. C_i(x) ≠ ⊥ ∧ C_j(x) ≠ ⊥ ⇒ C_i(x) = C_j(x). 즉, 전체 system에서 하나의 logical address에 대해 오직 하나의 unique valid value만 cache에 존재 가능 (cache coherence의 abstraction).

**Transition labels**:
- LStore_i(x,v), RStore_i(x,v), MStore_i(x,v), Load_i(x,v)
- LFlush_i(x), RFlush_i(x), GPF_i
- RMW_i(x,old,new) — 6종 (L-RMW, R-RMW, M-RMW × load-from-C vs load-from-M)
- Silent propagation τ
- Local crash E_i

**Operational rules** (Figure 2): 각 rule은 C와 M의 state transition을 수학적으로 정의. 예:
- LStore_i(x,v): C_i[x↦v], ∀j≠i. C_j[x↦⊥]
- RStore_i(x,v): C_k[x↦v] (x∈Loc_k), ∀j≠k. C_j[x↦⊥]
- MStore_i(x,v): M_k[x↦v] (x∈Loc_k), ∀j. C_j[x↦⊥]
- Load from C: C_j(x)=v≠⊥ ⇒ read v, C_i[x↦v] (copy to local cache)
- Load from M: ∀j. C_j(x)=⊥ ⇒ read from M_k(x)=v

**Crash step**: C_i = λx.⊥ (cache 소실), volatile memory는 0으로 reset, non-volatile memory는 보존.

**Proposition 1** (Rocq로 증명됨): RStore는 LStore보다 강함, MStore는 RStore보다 강함, RFlush는 LFlush보다 강함, owner에 대한 RStore와 LStore는 동등, RStore+LFlush로 RStore 시뮬레이션 가능, LStore+RFlush로 MStore 시뮬레이션 가능.

### 3. Litmus tests (Figure 3)

9개의 litmus test로 CXL0의 allowed/illegal behavior 검증. 주요 발견:

- **Test 1-3** (single machine): RStore 후 crash → value loss 가능 (✔). MStore 후 crash → loss 불가 (✗). LStore+LFlush 후 crash → loss 불가 (✗).
- **Test 4-5** (multi-machine): Remote NVMM에 LStore+LFlush → remote crash 시 loss 가능 (✔). RFlush 사용 시 loss 불가 (✗) — RFlush가 ∀j.C_j=⊥를 요구하므로.
- **Test 6-7**: Load가 remote cache의 value를 local cache에 복사 → 이후 다른 machine crash에도 value 보존.
- **Test 8-9** (multiple variables): RStore로 쓴 value를 다른 operation이 관찰한 후에도 loss 가능 (✔). MStore 사용 시 불가 (✗) — MStore가 durable order 보장.

## 핵심 기여

- **CXL0는 CXL 위의 concurrent program을 위한 최초의 formal programming model**. CXL specification의 복잡한 transaction들을 3-store, 2-flush abstraction으로 단순화.
- **Partial-system crash model에서 durable linearizability를 달성하는 최초의 general transformation** — FliT를 CXL0에 adaptation. 모든 linearizable object를 CXL0 위에서 durable하게 만듦.
- **CXL0는 현재와 미래의 다양한 CXL configuration을 포괄** — host-device pair부터 fully coherent shared pool까지.
- **향후 과제**: Asynchronous flush 지원, heterogeneous non-CXL privatized memory, posted/non-posted transaction 구분, relaxed durability semantics.

## 주요 결과

- **CXL0^LWB** (Load with implicit Write-Back): 모든 remote load가 memory로 write-back된 후에만 load 완료. Load-from-C 대신 C_i(x)=v≠⊥ ∧ ∀j.C_j(x)=⊥를 precondition으로 함. Most resilient.
- **CXL0^PSN** (cache line Poisoning on crash): Crash 시 crashed machine의 address가 모든 cache에서 ⊥으로 poison됨. Consecutive crash 간 inconsistency 방지.

FDR4 refinement checker로 세 variant 비교: CXL0가 가장 permissive, CXL0^LWB가 update loss에 가장 resilient, CXL0^PSN이 cross-crash consistency에 가장 강함.
