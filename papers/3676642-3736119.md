# HybridTier: an Adaptive and Lightweight CXL-Memory Tiering System

**Venue:** 
**저자:** Kevin Song, Jiacheng Yang (University of Toronto, Vector Institute), Zixuan Wang, Jishen Zhao (UC San Diego), Sihang Liu (University of Waterloo), Gennady Pekhimenko (University of Toronto, Vector Institute)

## 개요

현대 데이터센터는 폭발적인 메모리 수요에 직면해 있다. Meta는 rack당 TCO의 37%가 memory 비용이라고 보고한다. DRAM density 성장은 둔화되고 있으며, CXL 기반 memory tiering이 유망한 해결책으로 부상했다. CXL-attached memory는 local DRAM 대비 50-100 ns 추가 latency, 20-70% bandwidth를 가지므로, hot data를 fast-tier(local DRAM)에, cold data를 slow-tier(CXL)에 배치하는 tiering system이 필수적이다.

그러나 기존 tiering system은 세 가지 요구사항을 동시에 충족하지 못한다:

1. **Hot set 정확한 포착:** Frequency-based system (Memtis, HeMem)은 access count histogram으로 hot page를 정확히 식별하지만, EMA cooling의 lagging indicator 특성으로 hotness 변화에 느리게 적응한다. 실험에서 CacheLib workload의 hotness distribution 변화에 Memtis는 **1400초**가 소요되며, 이상적 적응 시간보다 1000초 느리다 (Figure 4). 반면 recency-based system (AutoNUMA, TPP)은 빠르게 적응하지만, cold page를 hot page로 오분류하여 fast-tier를 오염시킨다.

2. **변화하는 hotness에 빠른 적응:** Meta의 production cache는 10분만에 50%의 popular object가 더 이상 popular하지 않을 정도로 빠르게 변화한다. Graph analytics (Page Rank)와 ML training (XGBoost)에서도 5분만에 90%+, 50%+의 initially hot page가 cold로 전환된다 (Figure 2).

3. **최소 metadata overhead:** 1TB server에서 Memtis는 16B/page → 3.9GB metadata (0.39% overhead). 이 overhead는 AWS t2.nano $4.18/month × 100K servers 기준 연간 $31.8M 상당. 또한 PEBS sampling 시 page table/hash table lookup으로 인한 cache miss가 심각: Memtis는 L1 cache miss의 9%, LLC miss의 18%를 tiering activity가 차지한다 (Figure 5).

## 방법론

### 1. Dual-Metric Tracking: Frequency + Momentum

기존 시스템이 single metric만 사용해서 발생하는 tradeoff를 극복하기 위해, 각 page에 **두 개의 EMA metric**을 유지:

- **Access Frequency:** 긴 cooling period C (분~시간 단위). 장기적 hotness 분포 포착.
- **Access Momentum:** 짧은 cooling period C (초 단위). 최근 hotness 변화 감지.

#### Tiering Policy (Table 1)

| Frequency | Momentum | Action |
|-----------|----------|--------|
| High | High | Promote / No Action |
| High | Low | Promote / 2nd Chance (demotion 전 revisit) |
| Low | High | Promote / No Action (최근 hot해진 page 빠른 포착) |
| Low | Low | No Action / Demote |

- **Promotion:** frequency나 momentum 중 하나라도 threshold 초과 시 promote.
- **Demotion:** 둘 다 낮으면 즉시 demote. Frequency high, momentum low는 second chance 부여 (1분 후 revisit, access 없으면 demote).

Frequency threshold는 Memtis와 유사하게 hotness distribution + fast-tier capacity 기반 자동 조정. Momentum threshold는 empirically 하드코딩 (default=3).

**결과:** CacheLib workload 적응 시간이 Memtis의 1400초에서 **250초**로 단축 (Figure 4). AutoNUMA보다도 빠르고, steady-state latency도 가장 낮다.

### 2. Probabilistic Access Tracking via Counting Bloom Filter (CBF)

정확한(exact) access count가 tiering에 반드시 필요하지 않다는 관찰에 기반:

- **4-bit counters:** 최대값 15. Count ≥15인 page들은 모두 fast-tier에 배치해야 하므로 구별 불필요. 실제 모든 workload에서 frequency ≥15 page 비율이 ≤3% (social-graph 제외, Figure 16).
- **CBF 구조:** k=4 hash functions, M counters. GET = k개 counter 중 minimum 반환. INCREMENT = minimum counter들만 증가.
- **Memory 효율:** CBF size = f(fast-tier page 수, target error rate p=0.001). Fast-tier capacity에 비례하여 scale하므로, fast-tier가 작을수록 Memtis 대비 메모리 절감폭이 크다 (Table 4).

#### Blocked CBF: Cache Locality 최적화

Standard CBF는 k개 counter가 random access되어 worst case k cache miss 발생. **Blocked CBF**는 한 page의 k개 counter가 모두 동일한 64B cache line 내에 위치하도록 강제:

- 4-bit counter 기준: cache line당 128개 counter 수용.
- 단일 cache line access로 lookup 완료 → cache miss 1개 이하.
- Standard CBF 대비 false positive rate가 약간 증가하지만 성능 이점이 압도적 (Figure 14: bCBF가 cache miss를 추가 31-72% 감소).

### 3. Dual CBF Implementation

- **Frequency CBF:** 큰 capacity (e.g., 128MB). 느린 cooling.
- **Momentum CBF:** Frequency CBF의 1/128 크기. 빠른 cooling으로 대부분 page count가 빠르게 0으로 감소.

## 핵심 기여

**핵심 contribution:**

- **Frequency + Momentum dual tracking**으로 long-term hotness 정확도와 short-term 적응성을 동시에 달성. Single-metric 시스템의 근본적 tradeoff 극복.
- **Probabilistic CBF**가 memory tiering에 실용적임을 최초로 입증: 99.6%+ accuracy로 exact tracking을 대체 가능.
- **Blocked CBF**로 cache locality를 최적화하여 tiering cache overhead를 획기적 감소.

**정량적 성과:**
- Prior works 대비 최대 91% (geomean 19%) 성능 향상.
- 2.0-7.8× 적은 메모리 overhead, 1.7-3.5× 적은 cache miss.
- 3.2× 빠른 hotness 변화 적응.

**Broader significance:** CXL memory tiering의 실용적 배포를 가로막던 metadata overhead bottleneck을 해결. Probabilistic data structure의 tiering 적용은 향후 multi-tenant, global tiering으로의 확장 가능성을 열었다.

## 주요 결과

1. HybridTier runtime (`LD_PRELOAD`로 application에 투명하게 링크, userspace thread).
2. Intel PEBS로 memory access sampling → virtual address 수집.
3. 각 sample에 대해 frequency/momentum CBF 모두 UPDATE.
4. 100,000 samples batch로 promotion 결정 → `move_pages()` syscall로 migration.
5. Fast-tier free memory < `PROMO_WMARK` 시 demotion trigger. `/proc/PID/pagemaps` linear scan으로 cold page 식별.

## 평가

### Methodology

| 항목 | 내용 |
|------|------|
| Testbed | Dual-socket Intel Xeon 4314 (16-core ×2), 512GB DDR4/socket |
| CXL emulation | Remote NUMA node: idle latency 124ns, BW 34 GB/s |
| Fast:slow ratios | 1:16, 1:8, 1:4 (slow-tier 512GB fixed) |
| Baselines | AutoNUMA (MGLRU demotion), TPP, Memtis, ARC, TwoQ |
| Workloads | CacheLib (CDN, social-graph), GAPBS (BFS, CC, PR on Kronecker/uniform random graphs), SPEC CPU 2017 (603.bwaves, 654.roms), Silo/YCSB-C, XGBoost/Criteo |
| Page sizes | 4KB, 2MB THP |

### Overall Performance (geomean)

HybridTier vs. baselines (4KB pages):

| Baseline | Improvement (geomean) |
|----------|----------------------|
| TPP | 32% |
| AutoNUMA | 11% |
| Memtis | 29% |
| ARC | 50% |
| TwoQ | 40% |

### CacheLib (Figure 9)

- **Median latency:** TPP 대비 10%, AutoNUMA 대비 9%, Memtis 대비 **18%** 향상.
- **Throughput:** TPP 대비 15%, Memtis 대비 **23%** 향상.
- **효율성:** HybridTier 1:16이 다른 system의 1:8 성능을 능가 → 2× less fast-tier로 동등 성능.

### GAP (Figure 10)

- 최대 speedup: **BFS에서 33%** (두 번째 시스템 대비 평균).
- BFS는 "single-source" kernel로 iteration마다 hot set 변화 → HybridTier의 adaptivity가 가장 큰 효과 발휘.
- Uniform random graph는 Kronecker graph 대비 locality가 최악이나, HybridTier speedup이 15% → 53%로 증폭 (다양한 hotness 분포에 더 잘 대응).

### Detailed Comparison Against Memtis (Table 3, 4, Figure 13-14)

| Metric | Memtis | HybridTier | Reduction |
|--------|--------|------------|-----------|
| Adaptiveness (to steady-state) | 25-60+ min | 8.9-25.6 min | **3.2× faster** |
| Metadata memory overhead | 0.39% (constant) | 0.050-0.192% | **2.0-7.8× less** |
| Cache misses (L1) | 9% | 5% | **1.7× less** |
| Cache misses (LLC) | 18% | 4% | **3.5× less** (huge pages) |

### Huge Pages (Figure 12)

THP-aware Memtis 대비 hybrid-tier에서 9-11% 향상. THP에서도 frequency-momentum tracking의 이점 유지.

### CBF Accuracy (Table 5)

CBF vs. ground-truth hash table: 128MB CBF에서 **99.72%**, 64MB에서 **99.62%**, 8MB에서도 **96.92%** migration decision 일치.

### Sensitivity: Momentum Threshold (Figure 17)

- Threshold=3이 모든 workload에 optimal 근접.
- Threshold < 3: cold page 오 promotion으로 성능 저하.
- Threshold > 3: 추가 이득 미미.
- Social-graph는 CDN보다 threshold에 민감 (hot set이 커서 fast-tier 경쟁이 치열).

### All-Fast-Tier Baseline (Figure 11)

1:16, 1:8, 1:4에서 각각 14%, 9%, 6% 느린 성능 — near-optimal.

## 구현

- **Languages/LOC:** C++, 1,577 lines
- **배포 형태:** Userspace shared library (`LD_PRELOAD`)
- **의존성:** Intel PEBS (하드웨어), `move_pages()` syscall, `/proc` filesystem
- **Kernel modification 불필요**, application recompilation 불필요
- **Linux kernel v6.2**
- Huge page 지원: THP (madvise) 기반, frequency tracker 및 migration을 2MB 단위로 수행. 4KB 대비 metadata 128× 감소.
